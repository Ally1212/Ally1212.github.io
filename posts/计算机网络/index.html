计算机网络

一、概述

1.1 基本术语

  • 结点：网络中的结点可以是计算机、集线器、交换机或者路由器
  • 链路：从一个结点到另一个结点的物理线路，中间没有其他交点
  • 主机：连接在因特网上的主机
  • ISP：因特网提供商
  • IXP：互联网IXP的主要作用是允许两个网络直接相连并交换分组，而不需要再通过第三个网络来转发分组
  • RFC：“请求评议”，其中包含了Internet的重要文字资料
  • 广域网：任务是通过长距离运送数据
  • 域域网：将多个局域网相连
  • 局域网：学校或者企业大多拥有多个互联的局域网
  • 个人局域网PAN：把个人设备用无线技术连接起来的网络
  • 分组：因特网中的数据单元，由首部header和数据段组成。又称为包
  • 存储转发：路由器收到一个分组，先检查分组是否正确，并过滤掉冲突包错误。确定包正确后，查表，把包发送出去
  • 带宽：单位时间从一个点到另一个点所能通过的“最高数据率”，常用来表示网络传送数据的能力
  • 吞吐量：单位时间内通过网络的数据量
  • 按照网络的交换功能：可分为电路交换，报文交换，分组交换

1.3 基础知识

  1. 计算机网络是把许多计算机连接在一起，互联网是把许多网络连接在一起
  2. internet是互联网，Internet是全球最大的互联网
  3. 路由器：实现分组交换的，转发收到的分组。
  4. 分组交换采用存储转发，把一个报文分成几个分组后再转发，报文分成的等长的数据段，每一个数据段加上必要的控制信息，构成分组，又称为包，是互联网中的传送的数据单元
  5. 互联网按工作方式可以分为边缘部分和核心部分，主机在网络的边缘部分，作用是进行信息处理；核心部分（由大量网络和路由器组成），作用是提供联通性和交换
  6. 计算机通信是计算机中进程的通信。分为客户-服务器（C/S方式）、对等连接方式（P2P方式）
  7. 网络性能指标：速率、带宽、吞吐量、时延，信道利用率
  8. 网络协议：为网络通信而建立的规则，计算机各层和协议集合是网络的体系结构
  9. 五层体系：应用层、运输层、网络层、数据链路层、物理层
  10. 运输层主要协议：TCP和UDP；网络层主要协议：IP协议

二、物理层

2.1 基本术语

  • 数据：运输消息的实体
  • 信号：数据的电磁电气表现
  • 码元：在使用时间域的波形来表示数字信号时代表不同离散值的基本波形
  • 单工：只能有一个方向的通信而没有反方向的通信
  • 半双工：通信双方可以发送消息，但不能双方同时发送
  • 全双工：通信的双方可以同时发送和接受消息
  • 失真：失去真实性，收到的信号和发送的信号不同，有磨损和衰减
  • 奈氏准则：信道中，码元的传输是有上限的
  • 香农定律：在宽带受限且有噪声的信道中，为了不产生误差，信息的数据传输效率有上限值
  • 基带信号：来自信源的信号，没有经过调制的信号
  • 带通信号：把基带信号经过调制后，把信号的频率范围搬移到较高的频段以在信道中传输
  • 调制：对信号源的信息进行处理后加到载波信号上，使其在信道上传输
  • 信噪比：信号的平均功率和噪声的平均功率比
  • 信道复用：多个用户共享一个信道
  • 比特率：单位时间内传送的比特数
  • 波特率：单位时间内载波调制状态改变的次数
  • 复用：共享信道的方法
  • ADSL：非对称数字用户线
  • 光纤同轴混合网：在有限电视网的基础上开发的一种居民宽带接入网

2.2 基础知识

  1. 物理层的主要任务是确定与传输媒体接口有关的一些特性，如机械特性，电气特性，功能特性，过程特性
  2. 根据信号中代表消息的参数的取值方式不同，信号可以分为模拟信号和数字信号
  3. 来自信源的信号称为基带信号，信号在信道上传输要经过调制，调制有基带调制和带通调制
  4. 传输媒体可以分为两大类：导引型和非导引型

2.3 物理层主要做什么

物理层主要是透明的传输比特流，物理层考虑的是怎样才能在传输媒体上传输比特流，尽可能的屏蔽掉传输媒体和通信手段的差异，使上面的数据链路层感觉不到这些差异，就可以使数据链路层只考虑完成本层的协议和服务，而不必考虑网络的具体传输媒体和通信手段

2.4 常用的信道复用技术

  1. 频分复用：所有用户在同样的时间占用同样的带宽资源
  2. 时分复用：所有用户在不同的时间占用同样的频带带宽
  3. 码分复用：用户经过挑选的不同码型，各用户之间不会造成干扰，有抗干扰能力，不易被敌人发现
  4. 波分复用：就是光的频分复用

2.5 常见的宽带接入技术

  • ADSL：用非对称数据线ADSL电话线进行改造，不需要重新布线。ADSL的快速版本是高速用户数据线
  • HFC：光纤同轴混合网

三、数据链路层

3.1 基本术语

  • 链路：一个结点到相邻结点的一段物理链路
  • 数据链路：把协议的硬件和软件加到链路上就构成了数据链路
  • 循环冗余检验CRC：为了保证数据传输的可靠性，CRC是数据链路层广泛使用的一种检错技术
  • 帧：数据链路层的传输单元，由一个数据链路层首部和其携带的数据封装
  • MTU：最大传送单元，帧的数据长度有限
  • 误码率BER：一段时间内，传输错误的比特占所传输的比特总数的比率
  • PPP：点对点协议，即用户计算机和ISP进行通信时所使用的数据链路层协议
  • MAC地址：物理地址，硬件地址，用来定义网络设备的位置，在五层模型中，网络层负责IP地址，数据链路层负责MAC地址，一个主机有一个MAC地址，每一个网络位置有一个IP地址
  • 网桥：数据链路层用来实现中继，连接两个或多个局域网的网络互联设备
  • 交换机：通信系统中完成信息交换的设备，工作在数据链路层的交换机指的是交换式集线器，其实是一个多接口的网桥

3.2 基础知识

  1. 数据链路层主要是点对点信道和广播信道
  2. 数据单元是帧，数据链路层的三个问题：封装成帧、透明传输、差错检测
  3. 循环冗余检验CRC是一种检错方法，帧检验序列FCS是加在数据后面的冗余码
  4. PPP点对点协议是数据链路层使用最多的一种协议，特点是简单、只检测差错不纠正差错，不使用序号、不进行流量控制、可同时支持多种网络层协议
  5. PPPoE是为宽带上网主机使用的链路层协议
  6. 局域网的优点：广播功能，从一个站点可以方便的访问全网、便于系统的扩展和演变、提高系统的可靠性、可用性、可生存性
  7. 计算机与外部局域网通信需要通过通信适配器，又称为网络接口或者网卡，计算机的硬件地址就在适配器的ROM中
  8. 以太网采用无连接的工作方式，对发送的帧不进行编号，也不要求对方发回确认。目的站有差错就把他丢弃
  9. 以太网采用的协议是具有冲突检测的载波监听多点接入CSMA/CD，协议的特点是：发送前先监听，边发送边监听，一旦发现碰撞，立即停止发送，然后按退避算法等待一段时间再次发送
  10. 以太网的适配器具有过滤功能，只接受单播帧，广播帧和多播帧
  11. 使用集线器可以在物理层扩展以太网

四、网络层

4.1 基本术语

  • 虚电路：在两个终端设备之间，通过建立的双向的透明传输通道。虚电路表示这只是一条逻辑上的连接，发送分组
  • ARP： 地址解析协议，把IP地址转变为硬件地址
  • ICMP：网际控制报文协议（ICMP 允许主机或路由器报告差错）
  • 子网掩码：用来指明一个 IP 地址的哪些位标识的是主机所在的子网以及主机的位掩码。子网掩码不能单独存在，它必须结合 IP 地址一起使用。
  • CIDR：无分类域间路由选择 （特点是消除了传统的 A 类、B 类和 C 类地址以及划分子网的概念，并使用各种长度的“网络前缀”(network-prefix)来代替分类地址中的网络号和子网号）。
  • 默认路由：当在路由表中查不到能到达目的地址的路由时，路由器选择的路由。默认路由还可以减小路由表所占用的空间和搜索路由表所用的时间
  • 路由选择算法：路由选择协议的核心部分。因特网采用自适应的，分层次的路由选择协议。

4.2 基础知识

  1. TCP/IP 协议中的网络层向上只提供简单灵活的，无连接的，尽最大努力交付的数据报服务。网络层不提供服务质量的承诺，不保证分组交付的时限所传送的分组可能出错，丢失，重复和失序。进程之间通信的可靠性由运输层负责
  2. IP 数据报分为首部和数据两部分。首部的前一部分是固定长度，共 20 字节，是所有 IP 数据包必须具有的（源地址，目的地址，总长度等重要地段都固定在首部）。一些长度可变的可选字段固定在首部的后面。IP 首部中的生存时间给出了 IP 数据报在互联网中所能经过的最大路由器数。可防止 IP 数据报在互联网中无限制的兜圈子。
  3. 地址解析协议 ARP 把 IP 地址解析为硬件地址。ARP 的高速缓存可以大大减少网络上的通信量。因为这样可以使主机下次再与同样地址的主机通信时，可以直接从高速缓存中找到所需要的硬件地址而不需要再去以广播方式发送 ARP 请求分组
  4. 无分类域间路由选择 CIDR 是解决目前 IP 地址紧缺的一个好办法。CIDR 记法在 IP 地址后面加上斜线“/”，然后写上前缀所占的位数。前缀（或网络前缀）用来指明网络，前缀后面的部分是后缀，用来指明主机。CIDR 把前缀都相同的连续的 IP 地址组成一个“CIDR 地址块”，IP 地址分配都以 CIDR 地址块为单位
  5. 网际控制报文协议是 IP 层的协议。ICMP 报文作为 IP 数据报的数据，加上首部后组成 IP 数据报发送出去。使用 ICMP 数据报并不是为了实现可靠传输。ICMP 允许主机或路由器报告差错情况和提供有关异常情况的报告。ICMP 报文的种类有两种，即 ICMP 差错报告报文和 ICMP 询问报文
  6. 虚拟专用网络 VPN 利用公用的互联网作为本机构专用网之间的通信载体。VPN 内使用互联网的专用地址。一个 VPN 至少要有一个路由器具有合法的全球 IP 地址，这样才能和本系统的另一个 VPN 通过互联网进行通信。所有通过互联网传送的数据都需要加密

五、传输层

5.1 基本术语

  • 进程：指计算机中正在运行的程序实体
  • 传输层的复用与分用：复用指发送方不同的进程都可以通过同一个运输层协议传送数据、分用指接收方的运输层在剥去报文的首部后能把这些数据正确的交付到目的应用进程。
  • TCP：传输控制协议
  • UDP：用户数据报服务
  • 端口：端口的目的是为了确认对方机器的哪个进程在与自己进行交互
  • 停止等待协议：指发送方每发送完一个分组就停止发送，等待对方确认，在收到确认之后在发送下一个分组
  • 流量控制：就是让发送方的发送速率不要太快，既要让接收方来得及接收，也不要使网络发生拥塞
  • 拥塞控制：防止过多的数据注入到网络中

5.2 基础知识

  1. 运输层提供应用进程之间的逻辑通信，也就是说，运输层之间的通信并不是真正在两个运输层之间直接传输数据。运输层向应用层屏蔽了下面网络的细节（如网络拓补，所采用的路由选择协议等），它使应用进程之间看起来好像两个运输层实体之间有一条端到端的逻辑通信信道
  2. 网络层为主机提供逻辑通信，而运输层为应用进程之间提供端到端的逻辑通信
  3. UDP不需要建立连接，TCP需要建立连接
  4. UDP 的主要特点是 ① 无连接 ② 尽最大努力交付 ③ 面向报文 ④ 无拥塞控制 ⑤ 支持一对一，一对多，多对一和多对多的交互通信 ⑥ 首部开销小（只有四个字段：源端口，目的端口，长度和检验和）
  5. TCP 的主要特点是 ① 面向连接 ② 每一条 TCP 连接只能是一对一的 ③ 提供可靠交付 ④ 提供全双工通信 ⑤ 面向字节流
  6. TCP 用主机的 IP 地址加上主机上的端口号作为 TCP 连接的端点。这样的端点就叫做套接字（socket）或插口。套接字用（IP 地址：端口号）来表示。每一条 TCP 连接唯一地被通信两端的两个端点所确定。
  7. TCP 使用滑动窗口机制。发送窗口里面的序号表示允许发送的序号。发送窗口后沿的后面部分表示已发送且已收到确认，而发送窗口前沿的前面部分表示不允许发送。发送窗口后沿的变化情况有两种可能，即不动（没有收到新的确认）和前移（收到了新的确认）。发送窗口的前沿通常是不断向前移动的。一般来说，我们总是希望数据传输更快一些。但如果发送方把数据发送的过快，接收方就可能来不及接收，这就会造成数据的丢失。所谓流量控制就是让发送方的发送速率不要太快，要让接收方来得及接收
  8. 在某段时间，若对网络中某一资源的需求超过了该资源所能提供的可用部分，网络的性能就要变坏。这种情况就叫拥塞。拥塞控制就是为了防止过多的数据注入到网络中，这样就可以使网络中的路由器或链路不致过载。拥塞控制所要做的都有一个前提，就是网络能够承受现有的网络负荷。拥塞控制是一个全局性的过程，涉及到所有的主机，所有的路由器，以及与降低网络传输性能有关的所有因素。相反，流量控制往往是点对点通信量的控制，是个端到端的问题。流量控制所要做到的就是抑制发送端发送数据的速率，以便使接收端来得及接收
  9. 为了进行拥塞控制，TCP 发送方要维持一个拥塞窗口 cwnd 的状态变量。拥塞控制窗口的大小取决于网络的拥塞程度，并且动态变化。发送方让自己的发送窗口取为拥塞窗口和接收方的接受窗口中较小的一个
  10. TCP 的拥塞控制采用了四种算法，即慢开始，拥塞避免，快重传和快恢复。在网络层也可以使路由器采用适当的分组丢弃策略（如主动队列管理 AQM），以减少网络拥塞的发生
  11. 主动发起 TCP 连接建立的应用进程叫做客户，而被动等待连接建立的应用进程叫做服务器。TCP 连接采用三报文握手机制。服务器要确认用户的连接请求，然后客户要对服务器的确认进行确认
  12. TCP 的连接释放采用四报文握手机制。任何一方都可以在数据传送结束后发出连接释放的通知，待对方确认后进入半关闭状态。当另一方也没有数据再发送时，则发送连接释放通知，对方确认后就完全关闭了 TCP 连接

六、应用层

6.1 基本术语

  • 域名系统：
  • 域名系统（DNS）将人类可读的域名 (例如，www.baidu.com) 转换为机器可读的 IP 地址 (例如，220.181.38.148)。我们可以将其理解为专为互联网设计的电话薄。
  • 文件传输协议（FTP）：于 Internet 上的控制文件的双向传输，采用TCP可靠传输
  • 超文本传输协议：一种浏览器与服务器之间约定好的通信格式
